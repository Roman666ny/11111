## 2. Описание работы алгоритма

**Аналитика:**
- **Оператор `n % 10`** извлекает последнюю цифру числа
- **Оператор `n // 10`** (Python/Java) или `n / 10` (C++) отсекает последнюю цифру
- **Базовый случай:** когда число становится однозначным (`n < 10`), возвращаем само число
- **Рекурсивный случай:** складываем последнюю цифру с результатом рекурсивного вызова для оставшейся части

**Пример для n = 12345:**
```
sum_of_digits(12345)
= 5 + sum_of_digits(1234)    // 12345 % 10 = 5, 12345 // 10 = 1234
= 5 + 4 + sum_of_digits(123) // 1234 % 10 = 4, 1234 // 10 = 123
= 5 + 4 + 3 + sum_of_digits(12)
= 5 + 4 + 3 + 2 + sum_of_digits(1)
= 5 + 4 + 3 + 2 + 1 = 15
```

## 3. Определение временной сложности

**Временная сложность: O(d)**, где **d** — количество цифр в числе.

---

## 4. Почему такая временная сложность

- Алгоритм выполняет **ровно d рекурсивных вызовов**, где d - количество цифр
- На каждом шаге выполняются **константные операции O(1)**: 
  - сравнение `n < 10`
  - взятие остатка `n % 10`
  - целочисленное деление `n // 10`
  - сложение `+`
- Таким образом, **общее время работы = d × O(1) = O(d)**

**Математически:** для числа n количество цифр d = ⌊log₁₀n⌋ + 1, поэтому сложность можно записать как **O(log n)**.

## 5. Ответ на контрольный вопрос

**Вопрос 1:** *Что такое рекурсия и какие два обязательных компонента должна содержать рекурсивная функция?*

**Ответ:** 
Рекурсия — это метод программирования, при котором функция вызывает саму себя для решения задачи. Два обязательных компонента:
1. **Базовый случай** — условие завершения рекурсии, предотвращающее бесконечные вызовы
2. **Рекурсивный случай** — часть функции, где происходит вызов самой себя с изменёнными параметрами, приближающими решение к базовому случаю

**В нашем примере:**
- **Базовый случай:** `if n < 10: return n`
- **Рекурсивный случай:** `return n % 10 + sum_of_digits(n // 10)`
