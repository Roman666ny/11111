## 2. Описание работы алгоритма

**Аналитика:**
- Алгоритм использует оператор `%` (остаток от деления) для получения последней цифры числа
- Оператор `/` или `//` (целочисленное деление) используется для получения оставшейся части числа
- **Базовый случай:** когда число становится меньше 10 (одна цифра), функция возвращает само число
- **Рекурсивный случай:** последняя цифра (`n % 10`) прибавляется к результату вызова функции для оставшейся части (`n // 10`)

**Пример для n = 12345:**
```
sum_of_digits(12345)
= 5 + sum_of_digits(1234)    // n % 10 = 5, n // 10 = 1234
= 5 + 4 + sum_of_digits(123) // n % 10 = 4, n // 10 = 123
= 5 + 4 + 3 + sum_of_digits(12) // n % 10 = 3, n // 10 = 12
= 5 + 4 + 3 + 2 + sum_of_digits(1) // n % 10 = 2, n // 10 = 1
= 5 + 4 + 3 + 2 + 1 = 15 // базовый случай: n < 10, возвращаем 1
```

## 3. Определение временной сложности

**Временная сложность: O(d)**, где **d** — количество цифр в числе.

---

## 4. Почему такая временная сложность

- Каждый рекурсивный вызов обрабатывает одну цифру числа с помощью операций `%` и `//`
- Количество рекурсивных вызовов равно количеству цифр в числе
- На каждом уровне выполняется константное количество операций O(1)
- Общая сложность линейно зависит от количества цифр в числе

## 5. Ответ на контрольный вопрос

**Вопрос 1:** *Что такое рекурсия и какие два обязательных компонента должна содержать рекурсивная функция?*

**Ответ:** 
Рекурсия — это метод программирования, при котором функция вызывает саму себя для решения задачи. Два обязательных компонента:
1. **Базовый случай** — условие завершения рекурсии, предотвращающее бесконечные вызовы
2. **Рекурсивный случай** — часть функции, где происходит вызов самой себя с изменёнными параметрами, приближающими решение к базовому случаю

**В нашем примере:**
- **Базовый случай:** `if n < 10: return n`
- **Рекурсивный случай:** `return n % 10 + sum_of_digits(n // 10)`
